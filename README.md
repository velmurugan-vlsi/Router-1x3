# 1. Project Overview

The Router-1x3 is a packet-based digital router implemented entirely in
Verilog HDL. It receives packets from a single input interface and
forwards them to one of three output ports according to the two-bit
destination address contained in the packet header.

The design is partitioned into independent RTL modules: - FSM
Controller - Register - Synchronizer - Three FIFOs - Top Module

This modular architecture simplifies verification, improves readability,
and enables each functional block to be tested independently before
complete system integration.

# 3. Packet Format

                      Packet Structure

    +----------+----------------------+---------+
    |  Header  |  Payload (1-63 Bytes)| Parity  |
    +----------+----------------------+---------+
        8 bits         Variable          8 bits

## Header Format

                7                 2 1      0
              +--------------------+--------+
              | Payload Length[7:2]| DA[1:0]|
              +--------------------+--------+

### Destination Address (DA)

  DA   Destination
  ---- -----------------------
  00   FIFO0 / Output Port 0
  01   FIFO1 / Output Port 1
  10   FIFO2 / Output Port 2
  11   Invalid Destination

### Payload Length

Bits **\[7:2\]** specify the payload length in bytes (1--63 bytes).

### Parity Byte

The parity byte is generated by performing a bitwise XOR of the header
and every payload byte.

    Parity =
    Header
    ⊕ Payload Byte 1
    ⊕ Payload Byte 2
    ⊕ ...
    ⊕ Payload Byte N

The Register module computes the internal parity during packet
reception. After the final payload byte, the transmitted parity byte is
compared against the internally generated parity. If the two values
differ, the `err` signal is asserted.

# 4. Architecture

> **Insert Router Architecture Block Diagram Here**

The router architecture consists of one controller and one datapath.

**Datapath** - Packet enters through `data_in`. - Register captures the
packet. - Synchronizer decodes the destination. - One FIFO is
selected. - Packet is buffered. - Data is read through the selected
output port.

**Control Path** - FSM controls every stage of packet reception. - FIFO
status signals determine state transitions. - Synchronizer generates
FIFO write enables. - Register reports parity completion and errors.

# 5. Top Module

The top module integrates every RTL block into a complete router.

Responsibilities: - Interconnect all submodules. - Route control
signals. - Route packet data. - Connect FIFO status signals to the
FSM. - Export router outputs.

The modular organization keeps the controller, datapath, and buffering
logic independent while enabling seamless communication between them.

------------------------------------------------------------------------

# 6. FSM Module

The FSM is responsible for controlling the entire packet reception
process.

## DETECT_ADDRESS

The router remains in this idle state while waiting for a valid packet.
Once `pkt_valid` becomes active, the destination address is decoded and
the status of the selected FIFO is checked before packet reception
begins.

## LOAD_FIRST_DATA

The first byte of the packet (header) is accepted. The Register captures
the header, initializes parity generation, and prepares the selected
FIFO for data storage.

## LOAD_DATA

Payload bytes are accepted sequentially. The FSM enables writes to the
selected FIFO while continuously monitoring the FIFO full signal.

## FIFO_FULL_STATE

When the selected FIFO becomes full, packet transfer is temporarily
suspended. Incoming data is preserved until free space becomes
available.

## LOAD_AFTER_FULL

Once FIFO space is available again, the buffered byte is written and
packet reception continues without data loss.

## LOAD_PARITY

After the payload is complete, the parity byte is received and stored.

## CHECK_PARITY_ERROR

The internally generated parity is compared with the received parity
byte. If a mismatch exists, the `err` signal is asserted before
returning to the idle state.

------------------------------------------------------------------------

# 7. Register Module

The Register module forms the primary datapath.

Responsibilities: - Capture header. - Capture payload bytes. - Capture
parity byte. - Generate running parity. - Compare parity. - Buffer data
during FIFO full conditions. - Generate `parity_done`. - Generate
`low_pkt_valid`. - Generate `err`.

Internal storage: - Header Register - FIFO Full Register - Internal
Parity Register - Packet Parity Register

This module ensures that packets remain intact while moving through the
router.

------------------------------------------------------------------------

# 8. Synchronizer Module

The Synchronizer acts as the routing controller between the FSM and the
three FIFOs.

Responsibilities: - Decode destination address. - Generate one-hot
`write_enb[2:0]`. - Monitor FIFO full status. - Monitor FIFO empty
status. - Generate `vld_out_0`, `vld_out_1`, `vld_out_2`. - Generate
FIFO-specific soft reset signals.

Only one FIFO is enabled for writing during any packet transaction,
ensuring correct routing.

------------------------------------------------------------------------

# 9. FIFO Module

Each output port has an independent FIFO.

Responsibilities: - Temporary packet storage. - Independent read/write
operations. - Full and empty flag generation. - Soft reset support. -
Data delivery to output ports.

The FIFO preserves packet ordering (First-In, First-Out), guaranteeing
that packets leave the router in the same order they were received.
